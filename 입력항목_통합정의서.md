# 환급액 계산 입력항목 통합 정의서

> **문서 목적**: 경정청구 환급액 시뮬레이션을 위한 납세자 입력항목 전체 목록.
> 기존 `통합_환급액계산.md` [기본 정보 입력] 항목 + 누락 분석에서 도출된 34개 추가 항목을 통합 정리.
>
> **제외 범위**: 기준정보(세율표·공제율 등), 외부 법령정보, 첨부서류·증빙(비기능 요건)
>
> **필드 ID 체계**: `{카테고리}_{구분}{번호}` — CMN(공통), CRP(법인), INC(개인) / B(기존), M(누락추가)

## 6. 시뮬레이션 로직 참고사항 (Deep-research 보완)

> Gemini deep-research 분석 결과 (`deep-research-json-fields.md`) 기반 추가 고려사항

### 6.1 핵심 설계 원칙

1. **비교 구조 (Comparison Structure)**: 경정청구는 '당초 신고'와 '경정 청구'의 차액 소명이 핵심.
   주요 세액 필드는 `initial`/`corrected` 쌍으로 관리 → CMN_M011 그룹이 이를 담당.

2. **사원별 상세 데이터 확장**: deep-research에서 다음 추가 필드 권장:
   - `emp_type` (고용형태): REGULAR(정규직)/CONTRACT(계약직)/DAILY(일용직) → 1년 미만 계약직 배제
   - `monthly_salary` (월별 급여 배열): 최저임금 미만 월 판별용 (현재 CMN_M010e는 연간 합계만)
   - `social_ins_enroll` (4대보험 가입여부): 국민연금·건강보험 미가입자 상시근로자 배제
   - `contract_period_start/end` (근로계약 기간): 계약직 1년 미만자 배제 로직

3. **투자자산 확장 필드**: deep-research에서 추가 권장:
   - `is_used` (중고자산 여부): 중고자산은 통합투자세액공제 배제
   - `govt_subsidy_amt` (정부보조금): 취득가액에서 차감 후 공제 적용

4. **경정청구서 사유코드**: 홈택스 코드 체계 반영 (예: M01 세액공제 누락, M02 소득공제 누락 등)
